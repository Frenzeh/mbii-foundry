# Pet Control

`EAS_HI_PETCONTROL`

> Toggles control / command of the user's deployed pet.

## What it does

Activates the pet-control interface for owned summons (e.g. astromech, droideka companion, custom summon classes). Used to issue follow / hold / attack-target commands without having to swap weapon slots.

## Notes

- Added in R22.0.00.
- Requires `MB_ATT_PETCONTROL` on the class.
- Bind to `special1` / `special2` for in-combat use; the EAS entry just dispatches the action — actual command logic lives behind the attribute.
